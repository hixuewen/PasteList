# 变更：托盘新增设置菜单

## 为什么
当前托盘菜单中直接包含"开机启动"选项，随着功能增加，直接在托盘菜单中列出所有选项会导致菜单过长，影响用户体验。通过引入设置菜单，可以：
- 保持托盘菜单简洁有序
- 为未来更多设置选项提供扩展空间
- 改善用户交互体验

## 变更内容
- **新增**：托盘右键菜单中添加"设置"选项
- **新增**：创建设置窗口（SettingsWindow），包含开机启动等配置选项
- **迁移**：将开机启动功能从托盘菜单迁移到设置窗口
- **保留**：托盘菜单保留核心功能（显示窗口、设置、退出）

## 影响范围
- 受影响的规格：
  - tray: 系统托盘菜单和交互规范
  - startup: 开机启动管理功能规范
- 受影响的代码文件：
  - MainWindow.xaml: 托盘菜单定义需要修改
  - MainWindow.xaml.cs: 托盘菜单事件处理需要更新
  - 新增 SettingsWindow.xaml: 设置窗口UI定义
  - 新增 SettingsWindow.xaml.cs: 设置窗口逻辑
  - 新增 SettingsViewModel.cs: 设置窗口ViewModel
- 用户体验影响：
  - 用户需要多一步操作访问开机启动设置（点击设置 → 配置开机启动）
  - 但换来更清晰的菜单结构和更好的可扩展性

## 非目标
- 不更改开机启动的核心逻辑
- 不影响其他应用程序功能
- 不修改数据库结构或配置存储方式
